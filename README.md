## 🎯 Problem Statement

Over **63 million SMEs** in India lack access to formal credit due to:
- No credit history or thin CIBIL files
- Opaque, manual loan assessment processes
- Lenders unable to evaluate alternate financial signals (GST, bank flow)
- No explanation of why a loan was approved or rejected

**SMECreditAI** solves this with a transparent, AI-driven credit scoring system that uses **32 features** including GST compliance, bank statement patterns, and alternate signals — going far beyond a simple CIBIL score.

## 🤖 ML Model

### Dataset
- **1,500 synthetic SME records** with realistic Indian business distributions
- **70.6% creditworthy** base rate
- 6 feature categories: business profile, bank statement, GST, credit history, alternate signals, loan metrics

### 32 Features Used
```
Bank Statement  → cashflow_ratio, credit_debit_ratio, balance_to_emi_ratio ...
GST Data        → gst_compliance, gst_growth_rate, avg_gst_turnover ...
Credit History  → existing_credit_score, num_emi_bounces, previous_defaults ...
Business        → years_in_business, num_employees, industry_encoded ...
Alternate       → social_presence_score, online_reviews_rating, export_presence ...
Loan Metrics    → loan_to_turnover, debt_to_turnover, collateral_ratio ...
```

### Model Performance
| Metric | Score |
|---|---|
| Test Accuracy | **76–80%** |
| ROC-AUC | **0.84–0.85** |
| 5-Fold CV AUC | **0.843 ± 0.018** |
| Precision (Creditworthy) | **0.85** |
| Recall (Creditworthy) | **0.81** |

### Score Mapping (Non-linear piecewise)
```python
breakpoints = [
    (0.00, 300), (0.20, 450), (0.35, 550),
    (0.50, 640), (0.65, 700), (0.78, 760),
    (0.88, 820), (0.95, 860), (1.00, 900)
]
```

## 🚀 Local Setup

### Prerequisites
- Python 3.10+
- pip + virtualenv

### Steps

```bash
# 1. Clone
git clone https://github.com/yourusername/sme-credit-scoring.git
cd sme-credit-scoring

# 2. Virtual environment
python -m venv venv
venv\Scripts\activate        # Windows
source venv/bin/activate     # Mac/Linux

# 3. Install dependencies
pip install -r requirements.txt

# 4. Create .env file
echo SECRET_KEY=your-long-random-secret > .env

# 5. Generate dataset and train model
python ml/generate_dataset.py
python ml/train_model.py

# 6. Initialize database
python reset_db.py

# 7. Run
python run.py
```

Open **http://localhost:5000**
